# Retail Sales Performance Analysis (SQL & Tableau)

# Project Overview
This project demonstrates a full data analytics workflow: extracting raw retail data, processing it using Python and SQL, and delivering business insights through an interactive Tableau dashboard. 


# Business Objectives
- **Revenue Drivers:** Identify which products and categories generate the most profit.
- **Trend Analysis:** Visualize sales seasonality to optimize inventory levels.
- **Customer Insights:** Analyze purchasing patterns to improve marketing ROI.

# The Data Stack
- **Python (Pandas):** Used as the initial "Engine" to load and clean the raw CSV data.
- **SQL (SQLite):** Used to perform complex data aggregations and answer specific business questions.
- **Tableau:** Used to transform SQL results into a professional, interactive executive dashboard.
- **Google Colab:** The environment used for the technical analysis.

# Project Structure
- `Retail_Analysis.ipynb`: The Python notebook where I load the data, clean it with Pandas, and query it with SQL.
- `data/`: A folder containing the raw dataset (sourced from Kaggle).
- `README.md`: This document, containing project documentation and business insights.

# Key Questions Answered
1. What are the top 5 selling products by total revenue?
2. Which region or month shows the highest growth potential?
3. Are there specific days of the week where sales are underperforming?
